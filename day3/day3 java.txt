java architecture :
	*Class Loader : the .class read by jvm and store in method area. 
	
	*method area : ther is only one method area in jvm which is shared among all the class. this holds the class level info of each .class file
	
	*Heap : objects are allocated in this area.
	
	*stack : store temp variable and methods
	
	*PC Resgister : track which instruction has been executed . since instruction are executed by thread. each thread has a seperate pc registration
	
	*Native method area : A native method can access runtime data areas of the virtual machine
	
	*Native method interface: It enables java code to call or be called by native applications. native applications are program that are specific to the hardware and OS  of a system.
